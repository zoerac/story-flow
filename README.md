# StoryFlow Demo

StoryFlow Demo 是一个以“故事线”为核心的 AI-Native PPT 编辑器。

它把 PPT 的编辑起点从“逐页排版”前移到“叙事结构”：用户先通过对话明确演示意图，再选择主视觉、调整故事线，最后进入局部 AI 精修。

当前项目用于课程展示和交互验证，所有 AI 行为都由本地 mock 数据模拟，不接真实 API。

## 核心体验

StoryFlow 的完整动线分为四步：

1. 意图对齐：用户输入演示需求，AI 通过对话澄清听众、调性和重点，并生成故事线初稿。
2. 主视觉：从 Canva 风格候选和 AI 生成候选中选择视觉方向，决定初稿色板与页面表达基调。
3. 结构编辑：在故事线面板拖拽章节和子页，中央预览与 AI 对话同步响应，版本树自动记录改动。
4. AI 精修：围绕当前页、标题、正文或视觉素材提出局部修改意图，采纳后的结果写入版本树。

## 已实现功能

### 意图对齐

用户进入产品后先看到意图对齐页，而不是空白画布。用户可以用一句话描述本次演示需求，系统会继续询问听众、表达调性和汇报重点，并在右侧实时生成故事线草稿。

这一阶段还支持：

- 示例需求快速填充，降低首次输入成本。
- 听众与调性 chip 选择，模拟 AI 澄清需求的过程。
- 故事线草稿预览，用户可以在进入编辑器前先判断结构是否合理。
- 产品文档弹窗，用于在演示时解释 StoryFlow 的产品背景和设计逻辑。

### 主视觉选择

意图对齐完成后，用户不会直接进入结构编辑，而是先选择整套 PPT 的主视觉方向。当前提供 Canva 风格候选和 AI 生成候选两类视觉方案，所有候选都来自 mock 数据。

这一阶段已实现：

- `都看` / `Canva` / `AI生成` 三段式筛选。
- `都看` 模式下左右双栏展示，分别浏览 Canva 模板和 AI 生成候选。
- 单一来源模式下使用更紧凑的画廊布局，方便快速比较大量候选。
- 顶部视觉意图输入框，修改关键词后会刷新推荐排序。
- 候选卡片选中高亮、右侧详情预览和推荐理由说明。
- 点击“生成故事线与 PPT”后，把所选视觉映射到章节色板和初稿页面内容。

### 故事线结构编辑

结构编辑页是 Demo 的核心工作台。左侧是故事线，中央是当前页预览和 AI 对话，右侧是版本树。用户编辑的是章节和页面组成的叙事结构，幻灯片预览作为结构的渲染结果同步变化。

这一阶段已实现：

- 章节级拖拽重排，调整整份 PPT 的叙事顺序。
- 章节展开/收起，查看和选择章节下的子页。
- 子页选择与预览联动，中央预览会展示当前章节当前页。
- 章节合并，把一个章节并入另一个章节作为子页。
- 子页拆出，把某一页提升为新的一级章节。
- 结构变更后自动改写相邻章节的过渡文案，模拟 AI 对叙事上下文的理解。
- 左右面板可收起，主编辑区可聚焦展示。
- 分隔线可拖拽调整故事线、预览区和版本树宽度。

### AI 对话协作

AI 对话区以 mock 方式模拟结构协作能力。用户可以直接输入自然语言，也可以通过结构操作触发系统反馈。

当前支持：

- 普通建议回复，例如结构节奏、章节顺序和内容密度建议。
- 指令型对话解析，例如让 AI 调整结构、强化结论或压缩表达。
- 结构拖拽后的自动反馈，说明本次调整对叙事链条的影响。
- 将章节拖入对话区进行聚焦分析，获得针对该章节的建议。
- AI 操作后的撤销入口，可以回到操作前的版本节点。

### 版本树

版本树用于展示 StoryFlow 区别于普通线性撤销的核心能力：每次结构编辑和 AI 精修都可以形成可回溯的版本节点。

当前支持：

- 初始版本节点自动生成。
- 结构重排、章节合并、子页拆出、AI 改写等操作自动入树。
- 点击历史节点回溯，回溯后继续编辑会从该节点生成分叉。
- 手动保存当前版本，适合记录关键里程碑。
- 星标重要节点，星标节点受保护。
- 删除普通节点，并支持子节点上接父级的版本整理方式。

### AI 精修

AI 精修页用于模拟结构稳定后的单页打磨。用户可以围绕当前页面中的标题、正文、角标装饰或图片素材提出局部修改需求。

当前支持：

- 页面缩略条和大画布预览。
- 点击素材后在右侧显示对应编辑面板。
- 标题和正文文字编辑、字号调整、字体选择和文本框缩放。
- 图片素材候选切换、拖动和缩放。
- 角标装饰拖动和尺寸调整。
- 框选页面区域后生成 mock AI 方案。
- AI 方案按标题强化、正文压缩、视觉重绘等方向生成。
- 采纳方案后更新当前页内容，并可写入版本树。

## 技术栈

- Vite
- React
- JavaScript
- lucide-react
- CSS variables + inline styles

## 快速开始

```bash
npm install
npm run dev
```

默认开发地址：

```text
http://127.0.0.1:5173/
```

常用检查：

```bash
npm run build
npm run lint
```

## 目录结构

```text
src/
  App.jsx                         # 应用外壳、阶段路由、结构/视觉/精修页面
  main.jsx                        # React 入口
  components/
    Intro.jsx                     # 意图对齐入口
    IntroStoryline.jsx            # 意图阶段故事线预览
    DocModal.jsx                  # 产品文档弹窗
    StorylinePanel.jsx            # 故事线结构编辑
    SlidePreview.jsx              # AI 线预览组件
    ChatPanel.jsx                 # AI 对话与章节聚焦
    SelectionProposal.jsx         # 框选提案组件
    VersionTree.jsx               # 版本树
    Toolbar.jsx                   # 顶栏、阶段条与面板控制
  data/
    mock.js                       # 章节、视觉候选、AI 对话等 mock 数据
    intro.js                      # 意图对齐 mock 文案
    focus.js                      # 聚焦分析 mock
    proposals.js                  # 框选提案 mock
    doc.js                        # 产品文档内容
  hooks/
    useStoryflow.js               # 共享状态、结构操作和版本树 API
  lib/
    introEngine.js                # 意图对齐草稿生成逻辑
    refineEngine.js               # 结构/对话改写 mock 逻辑
  styles/
    tokens.css                    # 主题变量
    animations.css                # 过渡动画
```

静态视觉素材位于：

```text
public/canva_data/
canva_data/
```

## 数据模型

章节数据统一使用 `pages[]` 表示子页，页数由 `pages.length` 决定，不再使用 `slides` 计数字段。

```js
{
  id: "s1",
  title: "开场与背景",
  sub: "行业趋势与市场机会",
  c: "#7F77DD",
  bg: "#EEEDFE",
  bd: "#CECBF6",
  pages: [
    { id: "s1p1", h: "页标题", b: "页正文" },
  ],
}
```
